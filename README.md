# Beyond Boring: Battle of the Backlog

**Play it: https://kolatts.github.io/beyond-boring-battle-bots/**

A browser-based quiz battler for the *Agentic Coding for Total Beginners*
curriculum. BRILLIANT (you) faces BORING across 9 rounds — one per
curriculum module. Answer correctly and the Beam of Soul-Crushing Monotony
is returned to sender, with interest. Answer wrong and it lands on your
MORALE. Survive all 9 rounds and BORING — in his final form, the Change
Advisory Board — short-circuits into a pile of TPS reports.

Every question shows its full explanation whether you were right or wrong.
That is the entire point. Wrong answers are the highest-leverage teaching
moment in the game, and this is a study tool wearing a fighting-game
costume: losing a round just means retrying it.

## The 9 rounds

1. How LLMs Actually Behave
2. The Agentic Loop
3. Prompting & Instructions
4. Context Engineering
5. Project Memory & Instruction Files
6. Skills, Tools, MCP & Subagents
7. Verification & the Deep Loop
8. Safety, Review & Security
9. **BOSS:** GitHub Copilot in Practice

Tip: `?round=N` on the URL jumps straight to a round, if you want to study
one module without formally petitioning the board.

## Tech

- **Vite + TypeScript + Three.js.** The voxel robots reuse the model
  format from [beyond-boring-the-game](https://github.com/kolatts/beyond-boring-the-game)
  ("The Insurrection"): everything is box meshes, faces are canvas-texture
  CRT screens.
- Quiz UI, HP bars, and the state machine are plain DOM/CSS overlaid on
  the canvas — no framework, because a quiz does not need one.
- The full question bank (9 rounds × 10 questions, explanations included)
  lives in one editable file: [`src/data/questions.json`](src/data/questions.json).
  Each round draws its questions in shuffled order so replays differ.
- No backend. Deployed to GitHub Pages by GitHub Actions on every push
  to `main`.

```bash
npm install
npm run dev      # local dev at /beyond-boring-battle-bots/
npm run build    # typecheck + production bundle
```

## How this was made

Built end-to-end by **Claude Fable 5** running in Claude Code, from the
prompt in [`beyond-boring-quiz-battle-claude-code-prompt.md`](beyond-boring-quiz-battle-claude-code-prompt.md)
and the curriculum in [`curriculum.md`](curriculum.md). The 90-question
bank was generated by a 9-agent workflow — one subagent per curriculum
module, each grounded in `curriculum.md`, running in parallel (~2 minutes
wall clock, because parallelization is one of the five workflow patterns
and it would have been embarrassing not to use it).

Verification was done with `playwright-cli`: a scripted full playthrough
of all 9 rounds — exercising both the right- and wrong-answer paths,
asserting the explanation panel appears every single time, and finishing
with the boss KO and victory screen — run first against the local dev
server and then against the deployed GitHub Pages site. The retry-on-loss
path (10 wrong answers → morale zero → retry restores the round) was also
verified against production.

| 0 | Research: art direction, Insurrection voxel format, humor reference | ~1 min |

| 3 | Game build: robots, scene, state machine, UI + local smoke tests | ~14 min |
| 4 | Deploy: GitHub Actions → GitHub Pages, first successful deploy | ~1 min |
| 5 | Playwright verification: full runs vs. local + production, retry test | ~10 min |

Total: about 36 minutes from empty folder to verified production deploy,
which is less time than the average Change Advisory Board spends deciding
whether the meeting to schedule the review meeting has quorum.
